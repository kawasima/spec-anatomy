# SCREEN-BT-04 出張実績登録

## この画面の業務的役割

出張から帰社した申請者が、実際にかかった費用（実績費用）を入力して `behavior 出張実績を登録する` を呼び出す画面。入力対象は `事前承認OK` または `事前承認不要な出張申請` のどちらかの状態にある申請のみ。一覧画面から出張実績あり状態の行をクリックするか、申請者が自分の申請を直接開いて遷移してくる。

## 関連する behavior

- `behavior 出張実績を登録する` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))

この画面が表示・入力対象とする `data`。

- `data 事前承認OK` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))
- `data 事前承認不要な出張申請` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))
- `data 出張実績費用` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))

## 想定利用ユーザ

- 主アクター: 申請者（出張を実施した本人）
- 副アクター: なし

## 画面イベント

| イベント名   | 発生タイミング          | サーバ通信 | 対応する behavior / API                                                           | 正常時遷移先                  |
| ------------ | ----------------------- | ---------- | --------------------------------------------------------------------------------- | ----------------------------- |
| onRegister   | 登録ボタン onClick       | あり_同期  | `behavior 出張実績を登録する` / POST /api/v1/business-trips/{id}/actuals          | [SCREEN-BT-01](SCREEN-BT-01-business-trip-list.md)（登録完了トースト）|
| onBack       | 戻るボタン onClick       | なし       | −                                                                                 | [SCREEN-BT-01](SCREEN-BT-01-business-trip-list.md) |

### イベント詳細

#### onRegister

- バリデーション: 明細が1件以上あること。各明細の全項目が入力済みであること。実績日付が出張期間内であること。実績金額が0以上であること
- 呼び出す behavior / API: `behavior 出張実績を登録する` / POST /api/v1/business-trips/{id}/actuals
- 遷移条件:
  - 200 OK → SCREEN-BT-01 へ遷移し「実績を登録しました」とトースト
  - 400 ValidationError → 本画面に留まりバリデーションエラーを表示
  - 409 InvariantViolation → 「他の操作によって状態が変わりました」とトーストし再読み込み
  - サーバエラー → エラー画面へ

## 業務的事前条件・事後条件

- 事前条件: 対象申請が `事前承認OK` または `事前承認不要な出張申請` 状態であること。操作ユーザが申請者本人であること
- 事後条件: onRegister 完了後は `出張実績` 状態へ遷移

## アクセス制御

- 認証必須: はい
- 必要ロール: EMPLOYEE 以上
- 動的な表示制御: 申請者本人のみ操作可。他者の申請に URL 直打ちで到達しても登録ボタンは表示しない（または 403）

## 受け入れ基準

| ケース                   | 操作                                      | 期待結果                                                        |
| ------------------------ | ----------------------------------------- | --------------------------------------------------------------- |
| 正常系                   | 全必須項目を入力して登録する              | 出張実績状態に遷移し一覧に戻る。「実績を登録しました」トースト  |
| 業務エラー（明細0件）    | 明細を全削除してから登録する              | 「費用明細を1件以上入力してください」エラーが表示される          |
| 業務エラー（期間外日付） | 出張期間外の日付で明細を登録する          | 「出張期間内の日付を入力してください」エラーが表示される         |
| 状態不整合               | 既に実績登録済みの申請で登録ボタンを押す  | 「他の操作によって状態が変わりました」とトーストし再読み込み     |
