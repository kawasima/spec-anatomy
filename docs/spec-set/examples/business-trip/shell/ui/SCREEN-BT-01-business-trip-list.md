# SCREEN-BT-01 出張申請一覧

## この画面の業務的役割

申請者が自分の出張申請の状態を確認し、ドラフトの続きや実績登録へ進む入口となる画面。承認者は承認待ち（事前承認必要・出張実績あり）の申請を確認して詳細画面へ遷移する入口にもなる。ログイン後のトップ画面から遷移してくる。

## 関連する behavior

この画面が表示する `data`。

- `data 出張申請` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md)) — `出張申請ドラフト OR 申請済み出張申請` の全バリアント

## 想定利用ユーザ

- 主アクター（申請者）: 社員（全ロール）— 自分の申請一覧を見る
- 主アクター（承認者）: 社員（role >= MANAGER）— 承認待ち申請を見る

## 一覧表示

| 領域名       | ページング          | ソート条件                         |
| ------------ | ------------------- | ---------------------------------- |
| 申請一覧     | あり（20件/ページ） | 申請日時 降順（デフォルト）        |

### イベント詳細

#### onRowClick

クリックした申請の状態に応じて遷移先が変わる。

| 状態                   | 遷移先                                                                              |
| ---------------------- | ----------------------------------------------------------------------------------- |
| ドラフト               | [SCREEN-BT-03](SCREEN-BT-03-business-trip-create.md)（編集モード）                 |
| 事前承認必要           | [SCREEN-BT-02](business-trip-detail-screen.md)（承認者の場合）                     |
| 出張実績あり           | [SCREEN-BT-04](SCREEN-BT-04-business-trip-actuals.md)（申請者の場合）              |
| 出張実績あり           | [SCREEN-BT-05](SCREEN-BT-05-business-trip-final-approve.md)（承認者の場合）        |
| その他                 | 参照専用の詳細画面（このサンプルでは省略）                                          |

## 業務的事前条件・事後条件

- 事前条件: ログイン済みであること
- 事後条件: 各詳細画面へ遷移するか、新規申請作成画面へ遷移する

## 影響する集約・データ

- [`出張申請`](../../spec-model/business-trip.md) — 全バリアントを一覧に表示

## アクセス制御

- 認証必須: はい
- 必要ロール: 全ロール（EMPLOYEE 以上）
- 動的な表示制御: 申請者フィルタは MANAGER 以上のみ表示

## 受け入れ基準

| ケース               | 操作                                    | 期待結果                                                     |
| -------------------- | --------------------------------------- | ------------------------------------------------------------ |
| 正常系（申請者）     | ログイン後に一覧を開く                  | 自分の申請のみが一覧に表示される                             |
| 正常系（承認者）     | 承認者でログイン後に一覧を開く          | 申請者フィルタが表示され、全申請を検索できる                 |
| 新規申請             | 新規申請ボタンをクリック                | SCREEN-BT-03 へ遷移する                                      |
| 一覧行クリック       | ドラフト状態の行をクリック              | SCREEN-BT-03（編集モード）へ遷移する                         |
| 一覧行クリック       | 事前承認必要状態の行を承認者がクリック  | SCREEN-BT-02 へ遷移する                                      |

## 派生元参照のチェック

- すべての項目に派生元が書かれているか: ○（button は `−` と明示）
- Core にない項目を画面側で発明していないか: ○（状態ラベルは `data` バリアントの `kind` から派生）
- 派生計算が `<関数名>(<引数>)` の形式で書かれているか: ○（`sum(...)` を明示）
