## Core との対応

このコード値は、Core の `data` の OR 分岐の永続化を担う場合があります。

- 対応する Core の data: `data <data 名> = <バリアント1> OR <バリアント2> OR ...` ([../../../spec-model/data/<file>.md](../../../spec-model/data/<file>.md))

Core の OR 分岐をそのまま enum として永続化すると、各バリアントの状態依存項目が表現できません。状態依存項目がある場合は、テーブル側で NULL 可カラムとして表現し、Mapper で双方向に変換します（[table.md](table.md) 参照）。

状態依存項目のないシンプルな OR（例: `data 費用負担区分 = 自社負担 OR 先方負担`）は、enum として永続化すれば足ります。
