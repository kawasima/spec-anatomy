# SCREEN-BT-03 出張申請作成・編集

## この画面の業務的役割

申請者が出張予定（目的・期間・費用明細・費用負担区分）を入力し、出張申請ドラフトを作成する画面。ドラフト保存と申請提出の2つの操作を持つ。ドラフト保存では `出張申請ドラフト` を保持し、申請提出では `behavior 出張申請する` を呼び出して `申請済み出張申請` に遷移させる。一覧画面からドラフト行をクリックした場合は編集モードで同じ画面が開く。

## 関連する behavior

- `behavior 出張申請する` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))

この画面が表示・入力対象とする `data`。

- `data 出張予定` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))
- `data 出張申請ドラフト` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))

## 想定利用ユーザ

- 主アクター: 申請者（全ロール — 自分の申請を作成する）
- 副アクター: なし

## 画面イベント

| イベント名  | 発生タイミング                | サーバ通信 | 対応する behavior / API                                                      | 正常時遷移先                 |
| ----------- | ----------------------------- | ---------- | ---------------------------------------------------------------------------- | ---------------------------- |
| onSaveDraft | 下書き保存ボタン onClick       | あり_同期  | PUT /api/v1/business-trips/{id}（ドラフト保存）                              | 本画面（保存完了トースト）   |
| onSubmit    | 申請するボタン onClick         | あり_同期  | `behavior 出張申請する` / POST /api/v1/business-trips/{id}/submit            | [SCREEN-BT-01](SCREEN-BT-01-business-trip-list.md)（申請完了トースト）|
| onCancel    | キャンセルボタン onClick       | なし       | −                                                                            | [SCREEN-BT-01](SCREEN-BT-01-business-trip-list.md) |

### イベント詳細

#### onSaveDraft

- バリデーション: なし（ドラフトは未入力項目があっても保存可）
- 呼び出す API: PUT /api/v1/business-trips/{id}（新規作成時は POST /api/v1/business-trips）
- 遷移条件:
  - 200 OK → 本画面に留まり「下書き保存しました」とトースト
  - サーバエラー → エラー画面へ

#### onSubmit

- バリデーション: 必須項目（出張目的・出張開始日・出張終了日・費用負担区分・明細1件以上・各明細の全項目）がすべて入力済みであること。終了日 >= 開始日であること。予定金額が1以上であること
- 呼び出す behavior / API: `behavior 出張申請する` / POST /api/v1/business-trips/{id}/submit
- 遷移条件:
  - 200 OK → SCREEN-BT-01 へ遷移し「申請しました」とトースト
  - 400 ValidationError → 本画面に留まりバリデーションエラーを表示
  - サーバエラー → エラー画面へ

## 業務的事前条件・事後条件

- 事前条件: ログイン済みであること。編集モードの場合、対象申請が `出張申請ドラフト` 状態であること
- 事後条件: onSaveDraft 完了後は `出張申請ドラフト` 状態を保持。onSubmit 完了後は `申請済み出張申請` 状態へ遷移（`behavior 事前承認が必要か判断する` はサーバ側で連続して呼ばれ、`事前承認必要な出張申請` または `事前承認不要な出張申請` になる）

## 受け入れ基準

| ケース                   | 操作                                          | 期待結果                                                              |
| ------------------------ | --------------------------------------------- | --------------------------------------------------------------------- |
| 正常系（下書き保存）     | 必須項目未入力のまま下書き保存                | ドラフトが保存され「下書き保存しました」トーストが表示される           |
| 正常系（申請提出）       | 全必須項目を入力して申請する                  | 申請済みに遷移し一覧に戻る。「申請しました」トーストが表示される      |
| 業務エラー（バリデーション）| 出張目的を空のまま申請する                  | 「出張目的は必須です」エラーが表示され API を呼ばない                 |
| 業務エラー（日付順序）   | 終了日 < 開始日で申請する                     | 「終了日は開始日以降を指定してください」エラーが表示される            |
| 業務エラー（明細0件）    | 明細を全削除してから申請する                  | 「費用明細を1件以上入力してください」エラーが表示される               |
