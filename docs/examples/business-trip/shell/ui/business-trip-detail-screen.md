# SCREEN-BT-02 出張申請詳細画面（事前承認）

## この画面の業務的役割

販売部門マネージャー以上の承認者が、事前承認が必要と判定された出張申請の内容を確認し、承認または却下を行う画面。出張申請一覧画面（SCREEN-BT-01）から行クリックで遷移してくる。承認操作で `behavior 上長が事前承認する` を呼び出し、結果に応じて `事前承認OK` または `事前承認NG` に遷移させる。

## 関連する behavior

この画面が呼び出す Core の `behavior`。

- `behavior 上長が事前承認する` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))

この画面が表示する `data`。

- `data 事前承認必要な出張申請` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))
- `data 事前承認要件` ([../../spec-model/business-trip.md](../../spec-model/business-trip.md))

## 想定利用ユーザ

- 主アクター: 承認者（`data 承認者 = 社員` のうち role >= MANAGER）
- 副アクター: なし（申請者はこの画面で操作しない）

## 画面イベント

| イベント名 | 発生タイミング                         | サーバ通信 | 対応する behavior / API                                         | 正常時遷移先         |
| ---------- | -------------------------------------- | ---------- | --------------------------------------------------------------- | -------------------- |
| onApprove  | 承認ボタン onClick                     | あり_同期  | `behavior 上長が事前承認する` / [POST /api/v1/business-trips/{id}/pre-approve](../api/business-trip-api.md)（decision=approve） | 承認完了画面         |
| onReject   | 却下ボタン onClick + 却下理由確定後   | あり_同期  | `behavior 上長が事前承認する` / [POST /api/v1/business-trips/{id}/pre-approve](../api/business-trip-api.md)（decision=reject）  | 却下完了画面         |
| onBack     | 戻るボタン onClick                     | なし       | −                                                               | 出張申請一覧画面     |

### イベント詳細

#### onApprove

- バリデーション: なし（承認操作に追加入力なし）
- 呼び出す behavior / API: `behavior 上長が事前承認する`（decision=approve）
- 遷移条件:
  - 200 PreApproved → 承認完了画面へ
  - 409 InvariantViolation → 本画面に留まり「他の操作によって状態が変わりました」とトーストし再読み込み
  - 403 AuthorizationError → 本画面に留まり「承認権限がありません」とエラー表示
  - サーバエラー → エラー画面へ

#### onReject

- バリデーション: 却下理由が1文字以上500文字以下であること
- 呼び出す behavior / API: `behavior 上長が事前承認する`（decision=reject、rejectionReason=入力値）
- 遷移条件:
  - 200 PreApprovalRejected → 却下完了画面へ
  - 409 InvariantViolation → 本画面に留まり「他の操作によって状態が変わりました」とトーストし再読み込み
  - 403 AuthorizationError → 本画面に留まり「承認権限がありません」とエラー表示
  - サーバエラー → エラー画面へ

## 業務的事前条件・事後条件

- 事前条件: 表示対象の出張申請が `事前承認必要な出張申請` 状態であること。操作ユーザが申請者の上長（role >= MANAGER）であること
- 事後条件: onApprove 完了後は申請が `事前承認OK` 状態へ。onReject 完了後は `事前承認NG` 状態へ

## アクセス制御

- 認証必須: はい
- 必要ロール: SALES_VIEWER 以上（閲覧）、SALES_MANAGER 以上（承認・却下操作）
- 動的な表示制御: 操作領域の承認・却下ボタンの表示条件で個別に制御（上の表参照）

## 受け入れ基準

| ケース               | 操作                                              | 期待結果                                                 |
| -------------------- | ------------------------------------------------- | -------------------------------------------------------- |
| 正常系（承認）       | MANAGER が承認ボタンをクリック                    | 事前承認OK に遷移し、承認完了画面が表示される            |
| 正常系（却下）       | MANAGER が却下ボタンをクリックし却下理由を入力    | 事前承認NG に遷移し、却下完了画面が表示される            |
| 業務エラー（権限なし）| 上長でないユーザが承認ボタンをクリック            | 「承認権限がありません」が表示され、画面遷移しない        |
| 業務エラー（却下理由なし） | 却下ボタン押下後、却下理由を空のまま確定    | バリデーションエラーが表示され、API を呼ばない            |
| 状態不整合           | 別ユーザが先に承認済みの状態で承認ボタンをクリック | 「他の操作によって状態が変わりました」とトーストし再読み込み |
